Conclusions from profiling

The line profiler revealed that most time is spent on the kernel evaluation. This takes roughly 75%. Most of the remaining time is spent on computing differences.

A trick to reduce the \(nN\) kernel evaluations to a smaller number is binning.

(In the slides, \(n\) denotes the number of evaluation points for the estimate and \(N\) the length of the data vector.)

If \(N < n\) binning may not be beneficial.

Binning

Binning

The kernDens3 function uses bins around the grid points, where the density is evaluated, and counts the number of data points in each bin via the call to kernbin.

The kernbin function is a simple loop in R along the data vector.

You should note how arithmetic is used to determine which grid point is closest to a data point. This is much faster than a search along the grid points, and can be used when the grid points are equidistant.

Binning

kernbin
## function (x, lo, hi, n) 
## {
##     w <- numeric(n)
##     delta <- (hi - lo)/(n - 1)
##     for (i in seq_along(x)) {
##         ii <- floor((x[i] - lo)/delta + 0.5) + 1
##         w[ii] <- w[ii] + 1
##     }
##     w/sum(w)
## }

Benchmarking

x <- rnorm(2^13)
tmp <- microbenchmark(
  kernDens3(x[1:32], 0.2),
  kernDens3(x[1:64], 0.2),
  kernDens3(x[1:128], 0.2),
  kernDens3(x[1:264], 0.2),
  kernDens3(x[1:512], 0.2),
  kernDens3(x[1:1024], 0.2),
  kernDens3(x[1:2048], 0.2),
  kernDens3(x[1:4096], 0.2),
  kernDens3(x, 0.2)
  )
m <- 2^(5:13)
resKern3 <- cbind(m, aggregate(time ~ expr, tmp, median))

The computations become faster for long sequences when binning is used.

Benchmarking comparison
